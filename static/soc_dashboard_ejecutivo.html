Resumen ejecutivo

FarmWatch ya cuenta con una capacidad real de observabilidad y detección sobre infraestructura híbrida. La evidencia confirma ingestión de eventos de Cloud Run, Cloud SQL, GCP Audit, Windows, Active Directory, Raspberry Pi, Docker y el propio Wazuh Manager.

El SOC se clasifica en Nivel 3 — Definido: dispone de arquitectura, reglas personalizadas, correlación, severidad y mapeo MITRE ATT&CK; sin embargo, aún no opera de forma continua, no utiliza Active Response y no mide formalmente MTTD/MTTR mediante gestión de casos.

Decisión ejecutiva recomendada
Consolidar operación antes de ampliar alcance
Priorizar continuidad de agentes, MFA, gestión de incidentes y automatización controlada.

MTTD, MTTR y falsos positivos

MTTD

Σ(tiempo detección − inicio) / incidentes

Mide la rapidez para identificar una amenaza. En FarmWatch no existe aún un promedio formal; como ejemplo de laboratorio, una regla correlacionada puede detectar 5 fallos dentro de una ventana de 120 segundos.

Recomendación: almacenar first_seen, alert_time e ID de caso.

MTTR

Σ(tiempo contención − detección) / incidentes

Mide cuánto tarda el SOC en analizar y contener. Actualmente no es calculable porque no hay ticketing ni Active Response.

Recomendación: crear runbooks, SLAs y registro de cierre.

Falsos positivos

alertas descartadas / alertas investigadas × 100

Eventos genéricos de Rootcheck sobre varios binarios ilustran por qué se requiere validación antes de declarar compromiso.

Reducir con listas de exclusión justificadas, contexto por activo, umbrales y revisión mensual.

Recomendaciones del CISO

Hoja de ruta priorizada por reducción de riesgo y viabilidad.

1. Implementar MFA y fortalecer identidadCrítica
Aplicar MFA a administración, GCP, Wazuh y cuentas privilegiadas; eliminar credenciales compartidas.
2. Formalizar gestión de incidentesCrítica
Crear clasificación, propietarios, SLAs, evidencias y campos para MTTD/MTTR.
3. Mantener telemetría continuaAlta
Supervisar agentes, alertar desconexiones y diseñar operación 24/7 para una futura fase productiva.
4. Activar respuesta automatizada gradualAlta
Iniciar con bloqueo temporal de IP para fuerza bruta, sujeto a validación y lista segura.
5. Afinar reglas y falsos positivosMedia
Corregir ID duplicado 110023, revisar Rootcheck y medir tasa de descarte.
6. Avanzar hacia Zero Trust y SOAREvolutiva
Segmentar IoT, mínimo privilegio, identidad por servicio y playbooks automatizados.

Plan 30–60–90 días

Resultado esperado:
Transición de un SOC definido a un SOC gestionado con métricas, continuidad y respuesta repetible.

Conclusión ejecutiva

FarmWatch demuestra que una plataforma IoT académica puede incorporar controles de ciberseguridad de nivel profesional. La integración Wazuh–Google Cloud–Windows–IoT ya genera visibilidad y detecciones relevantes. El mayor valor creado es la capacidad de convertir eventos técnicos en señales priorizadas sobre amenazas a identidad, disponibilidad, integridad de datos y continuidad del monitoreo ganadero.

La siguiente inversión no debe enfocarse únicamente en crear más reglas, sino en operar mejor las existentes: continuidad, gestión de casos, MFA, afinación y automatización responsable.

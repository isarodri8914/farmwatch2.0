Recomendaciones del CISO

Ordenadas por prioridad de implementación
P1

Implementar MFA en accesos administrativos

Cloud Run, Cloud SQL y Active Directory hoy dependen de usuario/contraseña. Es la mitigación directa contra T1078 (Valid Accounts), la técnica con más incidentes confirmados de acceso.

Impacto esperado: -40% incidentes de credential access
P2

Automatizar respuesta (SOAR) para fuerza bruta y escaneos

Estos dos escenarios concentran el mayor volumen de alertas repetitivas. Bloquear automáticamente tras un umbral de intentos reduciría el MTTR sin intervención manual.

Impacto esperado: MTTR de 37 a <15 min en estos escenarios
P3

Ampliar detección en la capa IoT

Sensores (ESP32, MAX30100, GPS) son la categoría con más incidentes (27%) y el mayor riesgo residual (38/100). Se requieren reglas específicas para tráfico anómalo del dispositivo, no solo del backend.

Impacto esperado: reducir riesgo IoT de 38 a <25
P4

Cerrar la brecha de exfiltración de datos (T1041)

No existe hoy una regla que detecte transferencia anómala de datos hacia afuera de la infraestructura de Cloud SQL / Cloud Storage.

Impacto esperado: cobertura MITRE de 64% a ~71%
P5

Programa trimestral de reducción de falsos positivos

28.8% de las alertas escaladas no resultan en incidente real, consumiendo tiempo de analista que podría redirigirse a detección proactiva (threat hunting).

Impacto esperado: de ~29% a <15% de falsos positivos
